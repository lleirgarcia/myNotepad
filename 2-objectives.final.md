Acceptance criteria:

Todo side:
- User can add some texts to do, with priority and area.
- User will see the list of todos by different areas and by all areas together.
- If user adds calendar date to a todo it's gonna see it in the top of the list. 
- When user selects a todo it means is finish/ is done so it goes to the end of the list.

Notes side:
- User can add notes and a AI is gonna help to create todos (understanding the note and processing it).
- AI will process the notes and create the enough TODOs for the specific area user have.


Tech criteria:
- Add a DB to save all data.
- Integrate Open AI to understand the notes and processe it and create it.


Future criteria:
- User can use app in Android.
- User can log in/log out (multiuser).
